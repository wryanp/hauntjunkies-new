## üóÑÔ∏è Step 1: Set Up Supabase

### 1Ô∏è‚É£ Create a Supabase Project

| Step | Action |
|------|--------|
| 1 | Go to [supabase.com](https://supabase.com) and sign in |
| 2 | Click **"New Project"** |
| 3 | Choose your organization |
| 4 | Fill in project details |
| 5 | Click **"Create new project"** |

**Project Details:**

```yaml

```

‚è±Ô∏è **Wait ~2 minutes** for provisioning to complete

### 2Ô∏è‚É£ Run the Database Schema

<details>
<summary><strong>Step-by-step instructions (Click to expand)</strong></summary>

1. Once your project is created, navigate to **SQL Editor** (left sidebar)
2. Click **"New query"** button
3. Open the file `supabase-schema.sql` from your project
4. Copy the entire contents
5. Paste into the SQL Editor
6. Click **"Run"** (or press `Cmd+Enter` / `Ctrl+Enter`)

‚úÖ **Expected result:** `Success. No rows returned`

</details>

**What this creates:**

- ‚úÖ All database tables (reviews, comments, tickets, contact, etc.)
- ‚úÖ Row Level Security (RLS) policies
- ‚úÖ Indexes for performance
- ‚úÖ Triggers and constraints

### 3Ô∏è‚É£ Get Your Credentials

**Location:** Settings ‚Üí API

| Credential | Where to Find | Used For |

| **Project URL** | Under "Project URL" | `PUBLIC_SUPABASE_URL` |
| **anon/public key** | Under "Project API keys" | `PUBLIC_SUPABASE_ANON_KEY` |

üíæ **Save these** - you'll need them for Vercel environment variables

## üîç Troubleshooting

### Build Fails on Vercel

<table>
<tr>
<th>Symptom</th>
<th>Solution</th>
</tr>
<tr>
<td>Missing environment variables</td>
<td>Check all required vars are set in Vercel ‚Üí Settings ‚Üí Environment Variables</td>
</tr>
<tr>
<td>Type errors</td>
<td>Run <code>npm run check</code> locally to see errors</td>
</tr>
<tr>
<td>Node version mismatch</td>
<td>Ensure Node.js version is 20 or 22 in Vercel settings</td>
</tr>
</table>

### Database Errors

| Error | Cause | Fix |
|-------|-------|-----|
| "new row violates row-level security" | RLS policy blocking | Use service role key server-side |
| "relation does not exist" | Schema not run | Execute `supabase-schema.sql` |
| "column does not exist" | Schema mismatch | Re-run latest schema |

### Images Not Loading

<table>
<tr><td>‚úÖ Check Supabase Storage buckets are <strong>public</strong></td></tr>
<tr><td>‚úÖ Verify image URLs are complete (include full domain)</td></tr>
<tr><td>‚úÖ Check CORS settings in Supabase Storage</td></tr>
<tr><td>‚úÖ Ensure files uploaded successfully</td></tr>
</table>
